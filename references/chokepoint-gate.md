# Chokepoint Quick Filter

四问过滤用于防止主题看起来热，但公司无法捕获价值。

## 发散 → 取证 → 收敛（可选前置）

整体顺序遵循"发散 → 取证 → 收敛"，多模型只在两端做推理多样性，中间永远是联网/结构化取一手事实：

- **发散（可选，前置）**：仅深度研报、主题宽或不熟时，`python3 scripts/cross_verify.py --diverge "主题"` 让不同谱系模型各自提候选卡点节点、必须为真的离散事实和多空各一句。宽主题可显式拉长超时，例如 `--api-timeout 240 --cli-timeout 420 --max-tokens 3500`；若仍超时，保留成功模型输出并把缺位模型写入 QA。目的是扩大假设面、补主 agent 盲区，**塑造研究计划**（决定下一步该联网查什么）。
- **取证（必经）**：把发散产出的候选逐条用联网/`fetch_a_stock_data` 查一手来源。**护栏：模型提出的任何卡点进来都是 `待核验/线索级`，必须过下面四问 + 证据闸门、拿到一手证据后才能升级，没证据不进首页、不进核心。**
- **收敛（高风险末端）**：见 `cross-check.md`，用 `--redteam`/`--evidence` 对最终结论做反证与接地阅读。

约束：发散只跑一遍，不全量扫描；多家模型的重复假设由主 agent 去重聚类后再取证；不投票；缺密钥/工具时跳过发散，主 agent 自行拆链。

## 四问

| 维度 | 问题 | Yes 的证据 |
| --- | --- | --- |
| Demand | 需求是否真实发生或有可验证触发？ | 订单、招标、政策落地、客户 capex、财报指引、产品放量 |
| Transmission | 是否能传导到公司收入、利润、现金流或估值分类？ | 分部收入、价格、毛利、合同、客户验证、成本改善 |
| Bottleneck | 公司是否控制或贴近难替代、难扩张、低供应商数量的关键层？ | 认证周期、专利/工艺、客户锁定、产能稀缺、牌照/数据/渠道 |
| Elasticity | 小订单、客户认证、产能释放或估值重分类是否能带来足够弹性？ | 小基数、经营杠杆、估值折价、催化 3-12 个月内可观测 |

## 结论规则

- 0-1 个 Yes：观察名单或剔除，不进入推荐。
- 2 个 Yes：可标准研究，不强推。
- 3 个 Yes：可进入深度候选，需要 Evidence Gate 与 Red-team Gate。
- 4 个 Yes：可进入核心候选，但高风险时仍需 cross-check。

## 输出格式

```markdown
| 候选 | Demand | Transmission | Bottleneck | Elasticity | 结论 |
| --- | --- | --- | --- | --- | --- |
| 公司/环节 | Yes/No + 证据等级 | Yes/No + 证据等级 | Yes/No + 证据等级 | Yes/No + 证据等级 | 核心候选/弹性关注/观察/剔除 |
```

## 常见降级

- 需求真实，但公司只是概念映射：Transmission = No。
- 公司技术强，但商业化未落地：Transmission 或 Elasticity 降级。
- 公司控制瓶颈，但估值已完全透支：Elasticity 降级。
- 公司收入真实，但替代路线很快出现：Bottleneck 降级。

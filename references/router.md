# 请求路由

## 常见问法改写

用户可以用交易化语言提问，但输出必须改写为研究语言。

| 用户问法 | 默认转换 | 禁止输出 |
| --- | --- | --- |
| 能买吗 | 研究评级、证据强度、价格位置、失效条件 | 买入/建议买 |
| 什么时候买 | 观察区间、验证窗口、催化节点、价格位置 | 具体买点 |
| 卖出价是多少 | 目标价框架、上修/下修条件、证伪条件 | 卖出指令 |
| 仓位多少 | 风险等级、波动风险、组合暴露提醒 | 仓位比例 |
| 能不能无脑买 | 明确禁止无脑买，列出必须验证的问题 | 无脑买 |
| 哪个最值得看 | 评分排序、核心理由、最大反证 | 个性化交易动作 |
| 这条新闻利好谁 | 事件性质、核心价格变量、受益/受损链路和证据等级 | 直接交易建议 |

禁用话术只约束 skill 自身的指令性表达；来源引用、否定表达、合规说明和反例说明可以出现相关词，但必须明确不是本报告建议。

## 默认标的范围（A 股优先）

| 用户怎么说 | 主战场 | 其它市场 |
| --- | --- | --- |
| 默认（未指定市场） | 可投 A 股标的 | H 股、美股、全球赢家作次要对照 |
| “全球标的/全球/海外” | 全球赢家优先 | A 股、H 股作映射对照 |
| 指定“美股/US/某美股” | 该美股/美股市场为主 | 其它作对照 |
| 指定“港股/H 股” | 港股为主 | 其它作对照 |

规则：

- 默认以寻找可投 A 股标的为首要交付目标；首页结论默认 A 股候选在前，其它市场作对照行。
- 在用户可投范围内按市场分车道相对排序，选出该范围内相对最优的几家；即使它们全球排不进前列，也照常排序并老实标注（直接暴露 / 映射）。
- 全球这条标尺用于两件事：给次优标的定位（是映射不是主战场），以及剔除纯概念（沾边但无真实价值传导）。不是“只要有更优的美股就把整组 A 股降级”。
- 护栏不变：A 股没有真实暴露时直接写“暂无直接 A 股标的，最直接暴露在 X（美股/全球）”，不强行映射、不编造；纯概念无传导仍剔除。

## 中文入口与内部模式

| 中文入口 | 用户可能怎么说 | 内部模式 | 默认交付 |
| --- | --- | --- | --- |
| 事件快评 | 这条新闻利好谁、这个政策怎么看、突发事件怎么看 | 事件快评 | 短结论、受益/受损链路、证据等级、最大反证 |
| 财报后验证 | 财报出来了，原来的逻辑还成立吗 | 复盘验证 | 观点快照、关键数据变化、证伪状态 |
| 观点复盘 | 之前的 thesis 有没有被证伪 | 复盘验证 | 原始假设、现有证据、反证、下一验证点 |
| 催化日历 | 接下来要跟踪什么、未来两周看哪些验证点 | 复盘验证 | 催化验证短表 |
| 候选初筛 | 这个主题有哪些标的先看、哪个最值得看 | 深度瓶颈研报或轻量扫描 | 候选排序、筛除原因、下一步深挖对象 |
| 主题瓶颈扫描 | AI 服务器、碳化硅、液冷谁最卡 | 深度瓶颈研报 | 价值传导链、评分、证据闸门、正式交付 |

内部只保留 3 种模式：

- `事件快评`：先判断事件方向和核心价格变量，只输出当前判断、链路、证据等级和反证，不默认做 PDF。
- `复盘验证`：检查既有 thesis 是否仍成立，输出一次性复盘快照，不承诺跨会话自动维护状态。
- `深度瓶颈研报`：用户明确要求深度、正式交付、PDF、核心研究或投资标的分析时触发，走完整证据、评分、红队和 PDF QA。

## 模式契约

| 内部模式 | 默认强度 | 是否默认 PDF | 是否默认交付 QA | 首页/评分要求 |
| --- | --- | --- | --- | --- |
| 事件快评 | 轻量 | 否 | 否 | 短结论、证据等级、最大反证；不强制完整评分表 |
| 复盘验证 | 轻量到标准 | 否 | 否 | 复盘快照或催化短表；需要时给局部评分或证伪状态 |
| 深度瓶颈研报 | 标准到深度 | 是，正式交付时生成 | 是，正式交付时生成 | 正式 Markdown/PDF 必须有 `## 结论`、评分、证据等级、失效条件 |

`## 结论 + 评分分层 + PDF + 交付QA_YYYYMMDD.md` 是正式深度研报的重型契约，不强制套给事件快评和普通复盘。事件快评和复盘验证只有在用户明确要求“正式交付/PDF/保存版”时升级。

## 深度等级

| 深度 | 来源标准 | 输出形态 | 降级规则 |
| --- | --- | --- | --- |
| 轻量 | 3-8 个关键来源，或明确标为未验证初筛 | 4-8 段或短表 | 只能给观察/待验证，不给核心推荐 |
| 标准 | 10-25 个来源，覆盖关键结论 | Markdown 研报式回答 | 来源不足时减少候选，不凑数量 |
| 深度/PDF | 来源覆盖核心研究、反证、行情、估值、来源附录；数量随主题而定 | 中文 Markdown + 中文 PDF + `交付QA_YYYYMMDD.md` | 无法满足来源或交叉验证时，改为研究提纲/待验证版，默认只交付 Markdown |

来源数量是目标，不是硬凑指标。宁可交付 8 个高质量来源的降级结论，也不要用 25 个弱来源制造“看起来很深”的错觉。

## 自动升级与降级

自动升级到标准：

- 用户给出股票清单或要求排序。
- 需要判断真实收入、客户、订单、产能、估值。
- 轻量判断出现 3 个以上可投候选。

自动升级到深度/PDF：

- 用户要求 PDF、正式研报、可保存材料。
- 用户使用 `$bottleneck-scout-v3` 并要求“投资标的分析”“中国投资标的分析”“可投标的分析”“深度分析”“付费级分析”。
- 准备给 `核心研究`。
- 候选为微盘、低流动性、ST/退市边缘、港股小票、海外小盘。
- 结论依赖客户、订单、市场份额、产能、认证周期、政策审批或出口管制。

不要自动升级到深度/PDF的退出口：

- 用户明确说“快速”“简单”“初筛”“先聊聊”“不要 PDF”“只要结论”。
- 用户只是问方法、框架、某个单点事实，且没有要求正式交付。

如果用户措辞介于主题扫描和正式交付之间，优先按正式交付处理；宁可生成 Markdown+PDF，也不要把明显可保存的投研请求停在聊天式答案。

自动降级：

- 核心证据只有 KOL、媒体、截图或推断客户关系。
- 找不到最新行情、公告或一手披露。
- 交叉验证冲突未解决。
- 价值传导能讲通，但财务传导讲不通。
- 关键结论只能达到 `待核验/线索级` 或 `无支撑`。

## 输出长度控制

轻量模式只回答：

1. 当前判断。
2. 证据等级。
3. 最大反证。
4. 下一步最值得查什么。

标准和深度模式才展开完整报告骨架。

## 默认成本边界

- 默认单 agent 完成研究、写作和 QA。
- 用户明确要求深度时，优先加深检索、反证和 QA，不自动启动多 agent、多模型或全量数据源扫描。
- 多模型或多 agent 只用于离散高风险事实核验、明显证据冲突或不可信输入隔离，不作为核心研究候选的默认触发条件。

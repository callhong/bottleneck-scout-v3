# 价值传导链类型

先判断公司如何把 thesis 转成收入、利润、现金流或估值分类，再决定分析链条。硬件供应链只是其中一种。

## 硬件 / 半导体 / 制造

链条：

```text
需求 -> 系统 -> 部件 -> 工艺 -> 设备/材料 -> 产能 -> 收入/毛利
```

重点问题：

- 哪个部件或工艺最难扩产？
- 客户认证、良率、设备交期或材料纯度是否构成瓶颈？
- 公司是否有收入、订单、客户、产能或认证披露？

## 软件 / SaaS / 平台

链条：

```text
需求 -> 工作流痛点 -> 数据/分发/集成 -> 付费转化 -> 留存/毛利 -> 估值分类
```

重点问题：

- 用户工作流是否真的发生变化？
- 公司控制的是数据、分发、集成、生态入口，还是普通功能？
- 价值是否进入 ARR、净收入留存、毛利、获客成本或估值倍数？

## 医药 / 生物制造

链条：

```text
疾病/适应症 -> 临床证据 -> 注册路径 -> 生产/供应 -> 渠道/支付 -> 收入兑现
```

重点问题：

- 临床终点、注册路径和支付逻辑是否清晰？
- 工艺、产能、原料、冷链、渠道或医保是否是瓶颈？
- 单一适应症和平台技术的弹性要分开。

## 能源 / 公用事业

链条：

```text
负荷/政策需求 -> 并网/燃料/设备 -> 资本开支 -> 利用小时/电价 -> 现金流
```

重点问题：

- 需求是负荷增长、政策约束、价格机制，还是安全冗余？
- 瓶颈在设备、燃料、并网、审批、资本成本还是利用率？
- 收益是否受电价、补贴、容量电价或监管回报约束？

## 消费 / 品牌

链条：

```text
需求变化 -> 渠道触达 -> 供应/库存 -> 单店/客单/复购 -> 利润率
```

重点问题：

- 变化是短期爆品、渠道红利，还是复购和品类迁移？
- 瓶颈是产能、渠道、品牌心智、库存周转还是会员数据？
- 促销和库存会不会吞噬收入弹性？

## 金融 / 交易平台

链条：

```text
交易/资金需求 -> 牌照/风控/流动性 -> 费率/余额/杠杆 -> 收入质量
```

重点问题：

- 公司控制牌照、客户流量、资金成本、风控模型还是流动性？
- 收入是交易佣金、利差、管理费、支付费率还是一次性收益？
- 杠杆、监管、坏账和流动性是核心反证。

## 输出要求

每条链必须回答：

- 需求是否真实发生。
- 价值如何传导到公司财务。
- 哪个节点最难替代或扩张。
- 公司证据处于直接证据、交叉印证、框架推演、待核验/线索级还是无支撑。
- 替代路线或 design-out 风险在哪里。

## 拆解图适用性

适合画建议拆解图的场景：

- 技术路线、产业链、材料、设备、能源、AI 基础设施、国产替代、出口管制等有明确上下游传导的主题。
- 用户关心多层链路，例如需求、系统、器件、设备、材料、基础设施或公开标的映射。
- 需要比较不同环节的瓶颈稀缺度、成本占比、毛利率弹性、独占率、客户认证、扩产难度或价格变量。

不适合画图的场景：

- 单公司治理、财务修正、分红、减值、保险赔付等以财务事实核验为主的报告。
- 证据不足以支持具体链路，只能停留在待验证提纲。
- 价值传导不是上下游结构，而是政策、流动性、资产重估或事件博弈为主。

图层是可坍缩的，不是必填模板。最小结构为 `需求/事件 -> 瓶颈节点 -> 公开标的映射`；有可靠证据时再展开成本/利润池、竞争格局和证伪条件。

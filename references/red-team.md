# Red-team Gate

红队不是最后补几条风险，而是决定结论能否升级的闸门。

## 必查硬性反证

| 反证 | 问题 | 可能动作 |
| --- | --- | --- |
| design-out | 下一代架构是否能绕开这个节点？ | 降级瓶颈强度 |
| immaterial revenue | 节点真实但收入/毛利/现金流太小？ | 降级 alpha |
| 二供/替代路线 | 替代供应商或技术路线是否快速出现？ | 降级稀缺性 |
| 客户集中 | 逻辑依赖单一客户且未披露？ | 降级证据 |
| 稀释/债务 | 定增、可转债、ATM、现金流压力是否吞噬弹性？ | 降级评级 |
| 估值透支 | 证据是否无法吸收预期？ | 降级赔率 |
| 报表不落地 | 订单/认证/产能是否没有转成收入和毛利？ | 保留跟踪 |
| KOL 单点 | 关键线索是否只来自 KOL、截图或二级报道？ | 不得核心推荐 |

## 每个高优先级候选必须回答

- 最强的 3 个反方论点是什么？
- 哪条反证会让 thesis 立即失效？
- 需要看哪个公告、财报、招标、客户文件或价格信号？
- 当前估值是否已经买完全部好消息？
- 如果用户原始观点错误，错误在哪里？

## 输出格式

```markdown
| 候选 | 最强反方论点 | 证伪事件 | 当前处理 |
| --- | --- | --- | --- |
| 公司/环节 | ... | ... | 核心/降级/观察/剔除 |
```

反方论点要写得能真的打痛 thesis，不写“行业竞争加剧”这种空话。

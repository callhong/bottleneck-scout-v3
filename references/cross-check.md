# Cross-check Gate

交叉验证是高风险触发的可选闸门，不是每次默认执行。触发判断由 skill 自动完成，不交给用户凭经验判断。

## 自动触发条件

满足任一条件时，必须触发交叉验证，或在工具不可用时明确降级：

- 准备给 `核心推荐`、正式 PDF 结论或高弹性候选。
- 公司是微盘、低流动性、ST/退市边缘、港股小票、海外小盘。
- 关键证据来自 KOL、截图、二级报道或推断客户关系。
- 结论依赖客户、订单、市场份额、产能、认证周期等结构字段。
- 涉及稀释、定增、可转债、重大资产重组、政策审批、出口管制。
- 主模型发现“证据强度不够但结论很诱人”。

## 执行等级

| 风险 | 执行 |
| --- | --- |
| 低风险 | Codex 单独完成，保留来源库存 |
| 中风险 | Codex 自查 + 强制 Red-team Gate |
| 高风险 | 启动 1-3 个交叉验证 agent；若不可用，生成明确验证任务并降级结论 |

冲突未解决时，不做模型投票。按证据强度裁决：

```text
一手披露 > 客户/监管/交易所交叉验证 > 技术文档/专利/标准 > 可信媒体/行业报告 > KOL/社媒/截图
```

## Agent 分工

- Evidence Agent：只查一手证据，验证客户、订单、收入、产能、监管文件。
- Red-team Agent：专门找反证，攻击 design-out、二供、稀释、估值透支、报表不落地。
- Market/Valuation Agent：查行情、市值、估值、成交、拥挤、财务质量。

## 多模型独立核验工具（可选）

高风险闸门命中时，用多模型做推理多样性（不当事实判官）。三种模式，并行执行：

```bash
# 红队：对结论找最强反证、断点、必须用一手证据核验的清单
python3 scripts/cross_verify.py --redteam "公司X在卡点Y份额≥90%"

# 接地：只读联网/fetch_a_stock_data 取回的原文，判断是否支持结论，逐条引用
python3 scripts/cross_verify.py --evidence /tmp/snapshot.json "公司X披露该业务收入"

# 发散（瓶颈定位早期）：对主题提候选卡点 + 必须为真的离散事实 + 待验证项
python3 scripts/cross_verify.py --diverge "AI 服务器电源谁最卡"
```

- 顺序：**先联网/结构化取一手原文 → 再用模型接地阅读/红队**，模型永远在事实之后，不凭记忆断言时效性事实。
- 声音来源自动探测、默认尽量都用上、能降级：有 `config/opencode.env` 密钥 → DeepSeek/Qwen/MiniMax 面板；装了 Codex/Claude → 自动叠加兄弟 CLI（`claude -p`/`codex exec`）第二意见（`--no-cli` 可关）；都没有 → 打印待核验清单。任一模型失败或额度用尽自动跳过，不影响其余结果。
- **不投票**，只并列分歧；分歧表示"该回到一手证据"，最终按上面的证据强度链裁决。
- 仅高风险触发，不进入默认单 agent 路径；密钥文件 `config/opencode.env` 已 gitignore，不随仓库分发。

## 工具不可用时

如果无法启动 subagent、其他模型，或缺少 `config/opencode.env`（密钥未配置/失效/触顶限额）：

1. 写出交叉验证任务清单。
2. 将高风险结论降级为 `弹性关注`、`中性跟踪` 或 `待验证`。
3. 明确写出“未完成交叉验证，不能进入核心推荐”。
